# Slides

Strategic HTML presentation design with layouts and copywriting formulas.

## When to Use

- Marketing presentations and pitch decks
- Investor presentations
- Product launches and demos
- Data-driven slides with charts
- Strategic slide design

## Slide Structure

### Standard Presentation Flow

1. **Title Slide** - Brand, title, presenter name
2. **Problem/Context** - Set the stage, create urgency
3. **Solution** - Your proposition
4. **Features/Benefits** - Key points (2-4 slides)
5. **Evidence** - Data, testimonials, case studies
6. **Business Model** - How you make money (if applicable)
7. **Team** - Key people and credentials
8. **Call to Action** - Next steps
9. **Contact/Q&A** - Closing information

### Story Arc (Duarte Sparkline)

```
WHAT IS (Current State)
    |
    v
WHAT COULD BE (Future State)
    |
    v
WHAT IS (Problem)
    |
    v
WHAT COULD BE (Solution)
    |
    v
... (repeat pattern)
    |
    v
NEW BLISS (End State)
```

## Copywriting Formulas

### PAS (Problem-Agitate-Solution)

1. **Problem** - State the problem clearly
2. **Agitate** - Make it feel urgent and personal
3. **Solution** - Present your answer

### AIDA (Attention-Interest-Desire-Action)

1. **Attention** - Hook with headline
2. **Interest** - Build curiosity
3. **Desire** - Show benefits
4. **Action** - Clear CTA

### FAB (Features-Advantages-Benefits)

1. **Features** - What it does
2. **Advantages** - How it helps
3. **Benefits** - Why it matters

### STAR (Situation-Task-Action-Result)

1. **Situation** - Context
2. **Task** - Challenge
3. **Action** - What you did
4. **Result** - Outcome

## Typography Guidelines

### Font Sizes (for 1920x1080 slides)

| Element | Size Range |
|---------|------------|
| Title | 44-60pt |
| Subtitle | 24-32pt |
| Body | 18-24pt |
| Captions | 14-16pt |

### Hierarchy Rules

1. **One idea per slide** - Don't overcrowd
2. **6x6 rule** - Max 6 lines, 6 words per line
3. **Contrast** - Title > Subtitle > Body
4. **Alignment** - Consistent left/center alignment

## Color Guidelines

### Slide Color Roles

| Role | Purpose |
|------|---------|
| Background | Primary canvas color |
| Text | High contrast readability |
| Accent | Highlights, CTAs |
| Secondary | Supporting elements |

### Contrast Requirements

| Element | Minimum Ratio |
|---------|---------------|
| Body text | 4.5:1 |
| Large text | 3:1 |
| UI elements | 3:1 |

## Animation Guidelines

### Entrance Animations

| Type | Use Case | Duration |
|------|----------|----------|
| Fade | Subtle, professional | 300-500ms |
| Slide | Directional flow | 400-600ms |
| Scale | Emphasis | 200-400ms |
| Wipe | Progressive reveal | 500-800ms |

### Animation Rules

1. **Purpose** - Every animation should have meaning
2. **Consistency** - Use same animation style throughout
3. **Timing** - Keep under 1 second
4. **Reduced motion** - Respect user preferences

## Best Practices

### Content

- [ ] One main idea per slide
- [ ] Clear headline that states the point
- [ ] Supporting visuals, not walls of text
- [ ] Data visualized, not raw numbers
- [ ] Consistent terminology

### Design

- [ ] Consistent alignment and spacing
- [ ] Brand colors applied
- [ ] High contrast text
- [ ] Professional imagery
- [ ] Clean, uncluttered layout

### Delivery

- [ ] Logical flow from start to finish
- [ ] Clear transitions between topics
- [ ] Strong opening and closing
- [ ] CTA clear and actionable
- [ ] Contact information visible

## Common Mistakes to Avoid

1. **Too much text** - Use bullet points, not paragraphs
2. **Low contrast** - Ensure text is readable
3. **Inconsistent design** - Maintain visual coherence
4. **No clear CTA** - Tell audience what to do next
5. **Poor images** - Use high-quality, relevant visuals
6. **Too many slides** - Be concise, edit ruthlessly
7. **No story** - Build narrative, not just information
